An API playground.

Think small. Build mutations with everyone.

This project is an attempt to encourage developers of all skills to build small, focused services with only two API calls - one to accept input and one to display the final output.

The specs

Specifications can be viewed in spec.html

Current script flow:

  1. Grab token displayed on your screen.
  2. Add 1-3 services to your script.
  3. Play! Or share output however you want.

What kind of services can be built?

Imagine submitting the text 'hello meatspacers' to three external services - one capitalizes the text, the next one embeds the capitalized text into an animated gif and the next one sends it out to a Twitter account. The script path would be:

  1. change 'hello meatspacers' to 'HELLO MEATSPACERS'
  2. change 'HELLO MEATSPACERS' to an animated gif that includes the words
  3. share a link to the animated gif from a registered Twitter account
